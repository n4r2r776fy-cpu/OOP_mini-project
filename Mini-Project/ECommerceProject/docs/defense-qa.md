# Питання для захисту та короткі відповіді

1. Які рефакторинги були найважливішими перед релізом?
- Усунення дублювання обчислення суми, стандартизація поведінки `ReduceStock`.

2. Чим `FINAL_REPORT.md` відрізняється від `README.md`?
- README — короткий технічний вступ і інструкції запуску; FINAL_REPORT — глибокий технічний аналіз рішень і компромісів.

3. Які теми курсу проект демонструє найкраще?
- Тестування, LINQ/колекції, базові патерни (Strategy), SOLID-підходи у структуруванні сервісів.

4. Яке архітектурне рішення ви б змінили при початку проєкту заново?
- Використати легковагу базу даних або in-memory database замість JSON persistence для масштабованості.

5. Як довести, що проєкт готовий до захисту?
- Усі критичні тести проходять, документація відповідає коду, є демо-сценарій та реліз-план.
